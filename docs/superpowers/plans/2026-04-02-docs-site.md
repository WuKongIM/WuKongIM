# Docs Site Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create a new root-level `docs-site/` app that uses Fumadocs on a Vite-based React Router stack, serves WuKongIM docs from `/`, and builds static assets into `docs-site/dist/`.

**Architecture:** Start from the official Fumadocs `react-router-spa` template, then trim it down to a minimal documentation shell. Replace the default `/docs` route model with a root-mounted docs experience so `/` renders the docs homepage and `/getting-started` renders a placeholder page. Use command-based smoke checks instead of unit tests because this work is primarily framework scaffold and configuration.

**Tech Stack:** Vite, React 19, React Router 7, Fumadocs, fumadocs-mdx, TypeScript, Tailwind CSS 4

---

- [ ] **Step 2: Replace the route table with a minimal docs-only route**

Update `docs-site/app/routes.ts` to keep only the docs catch-all:

```ts
import { route, type RouteConfig } from '@react-router/dev/routes';

export default [route('*', 'routes/docs.tsx')] satisfies RouteConfig;
```

- [ ] **Step 3: Move the content base URL from `/docs` to `/`**

Update `docs-site/app/lib/shared.ts`:

```ts
export const appName = 'WuKongIM Docs';
export const docsRoute = '/';
export const docsContentRoute = '/llms.mdx/docs';
```

Update `docs-site/app/lib/source.ts` so `loader({ baseUrl: docsRoute })` uses `/`.

- [ ] **Step 4: Make prerendering generate root-based pages**

Update `docs-site/react-router.config.ts` so:

```ts
import type { Config } from '@react-router/dev/config';
import { glob } from 'node:fs/promises';
import { getSlugs } from 'fumadocs-core/source';

function toDocUrl(slugs: string[]) {
  return slugs.length === 0 ? '/' : `/${slugs.join('/')}`;
}
```

The `prerender()` implementation should:
- include `getStaticPaths()`
- glob `content/docs/**/*.mdx`
- convert each MDX file into either `/` or `/<slug>`
- keep the markdown export path `/llms.mdx/docs/...`

- [ ] **Step 5: Make the docs page handle the root path**

Update `docs-site/app/routes/docs.tsx` so the loader can handle an empty `params['*']`:

```ts
const slugs = (params['*'] ?? '').split('/').filter(Boolean);
const page = source.getPage(slugs);
```

Keep the existing `DocsLayout`/`DocsPage` rendering pattern, but remove GitHub and search-only chrome if it is not needed for the first pass.

- [ ] **Step 6: Remove template-only extras**

Delete the unused home, search, and llms files listed above, and remove related imports from `docs-site/app/root.tsx`.

- [ ] **Step 7: Verify the root-mounted site still builds**

Run: `yarn --cwd docs-site build`
Expected: build succeeds and generated routes now target `/` and root-level doc slugs

- [ ] **Step 8: Commit the route-model conversion**

Run:

```bash

git commit -m "feat: mount docs-site pages at the site root"
```
