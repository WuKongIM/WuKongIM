# Web I18n Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add extensible front-end internationalization to `web/` with `react-intl`, supporting `zh-CN` and `en`, browser-language defaults, persisted user selection, login/topbar switchers, and locale-aware UI text plus timestamp formatting.

**Architecture:** Introduce a focused `web/src/i18n/` slice that owns locale detection, persistence, message bundles, and the `IntlProvider` bridge. Keep the rest of the app consuming translated message IDs and locale-aware formatting helpers rather than writing language branches inline, then migrate shared shell text first and page-level text second.

**Tech Stack:** React 19, React Router 7, TypeScript, Vite, Vitest, Testing Library, `react-intl`, existing `Zustand` auth store, browser `localStorage`.

## Critical Review Notes Before Starting

- Do not hide locale state inside the auth store. Language preference must survive logout and stay available on `/login`.
- Do not keep `navigation.ts` as English display text. Move it to message IDs so `SidebarNav` and `Topbar` share one metadata source.
- Do not leave raw `toLocaleString()` calls tied to browser defaults. Wrap them in locale-aware formatting that reads the active app locale.
- Avoid a partial migration where shell chrome is translated but shared manager components still emit English fallback text like `Retry` or `Loading manager data.`

```ts
export type AppLocale = "en" | "zh-CN"

export const SUPPORTED_LOCALES = ["en", "zh-CN"] as const
export const DEFAULT_LOCALE: AppLocale = "en"
export const LOCALE_STORAGE_KEY = "wukongim_manager_locale"

export function normalizeLocale(input: string | null | undefined): AppLocale {
  const value = (input ?? "").toLowerCase()
  if (value.startsWith("zh")) {
    return "zh-CN"
  }
  if (value.startsWith("en")) {
    return "en"
  }
  return DEFAULT_LOCALE
}
```

```tsx
export function LocaleSwitcher() {
  const { locale, setLocale } = useLocale()

  return (
    <div aria-label="Language switcher" className="inline-flex gap-1">
      <Button
        aria-pressed={locale === "zh-CN"}
        onClick={() => setLocale("zh-CN")}
        size="sm"
        variant={locale === "zh-CN" ? "default" : "outline"}
      >
```
