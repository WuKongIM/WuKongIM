# Web 管理后台重设计

## 概述

WuKongIM 管理后台全面重设计，解决当前导航项过多（19 个平铺菜单）和分组逻辑不直观的问题。目标用户包括运维工程师和业务开发者。

## 设计决策

### 导航架构：顶部主导航 + 侧边栏子菜单

顶部横向放 4 个主区域 Tab，选中后侧边栏只显示该区域的子页面（每个区域 2-6 项）。

**顶部 Tab：**
- 总览
- 集群运维
- 业务管理
- 系统

### 页面分组

**总览（2 页）：**
- 仪表盘 — 核心指标与集群健康
- 实时监控 — 吞吐量、连接、延迟

**集群运维（6 页）：**
- 节点 — 节点清单与生命周期（含扩容操作面板）
- 槽位 — 分配与运行时状态
- 频道集群 — ISR 健康与副本（内含 Tab：总览/列表/异常）
- 任务 — 协调与迁移任务
- 拓扑 — 副本关系全景
- 诊断 — 消息追踪、网络、日志查看器（内含 Tab：追踪/网络/控制面日志/槽位日志）

**业务管理（4 页）：**
- 用户 — 用户列表与在线状态
- 频道 — 创建、订阅者、黑白名单
- 消息 — 按频道查询消息
- 系统用户 — 机器人与通知账号

**系统（3 页）：**
- 权限 — 管理员与角色配置
- Webhook — 事件回调配置
- 连接 — 当前连接清单

### 合并与降级

| 原页面 | 处理方式 |
|--------|----------|
| 扩容（onboarding） | 合并到「节点」页面的操作面板 |
| 控制面日志 | 合并为「诊断」页面的子 Tab |
| 槽位日志 | 合并为「诊断」页面的子 Tab |
| 网络 | 合并为「诊断」页面的子 Tab |
| 频道集群列表 | 合并为「频道集群」页面内 Tab |
| 异常频道 | 合并为「频道集群」页面内 Tab |

### 视觉风格：工程监控风

**字体：**
- 数字、状态、代码：Monospace（Geist Mono）
- 正文、标签：Geist Sans

**颜色系统：**
- 背景：#09090b（主）、#0a0a0a（卡片）、#1a1a1a（边框）
- 文字：#fafafa（主）、#a1a1aa（次）、#525252（辅助）
- 状态语义：绿 #22c55e = 健康、蓝 #3b82f6 = 运行中、黄 #eab308 = 警告/重试、红 #ef4444 = 错误/丢失

**设计语言：**
- 状态指示灯（6px 圆点）+ 大号 Monospace 数字
- 紧凑数据密度，表格行高适中
- 卡片用 1px solid 边框，无阴影
- 面包屑路径用 uppercase monospace 显示当前位置

### 交互细节

- 顶部 Tab 切换时侧边栏内容跟随变化，无页面刷新
- 侧边栏活跃项：深色背景 + 左侧 2px 绿色边框
- 页面头部显示 uppercase monospace 路径（如 `CLUSTER / NODES`）
- 指标卡片左上角状态灯实时反映健康状态
- 表格支持排序和筛选，保持 monospace 数字对齐

## 技术约束

- 保持现有技术栈：React 19 + React Router + Tailwind CSS 4 + shadcn + zustand
- 保持 i18n 支持（react-intl）
- 路由结构需要调整以支持顶部 Tab 对应的 layout 嵌套
- 新增 Geist Mono 字体（项目已有 Geist Sans）

```

/dashboard                 → 总览 > 仪表盘
/monitor                   → 总览 > 实时监控
/cluster/nodes             → 集群运维 > 节点
/cluster/slots             → 集群运维 > 槽位
/cluster/channels          → 集群运维 > 频道集群
/cluster/tasks             → 集群运维 > 任务
/cluster/topology          → 集群运维 > 拓扑
/cluster/diagnostics       → 集群运维 > 诊断
/business/users            → 业务管理 > 用户
/business/channels         → 业务管理 > 频道
/business/messages         → 业务管理 > 消息
/business/system-users     → 业务管理 > 系统用户
/system/permissions        → 系统 > 权限

```

旧路由通过 `<Navigate replace>` 重定向到新路径。
