这页只收最常见、最容易卡住接入的共性问题。

如果你现在遇到的是“就是连不上”或“消息收发不对”，先看这里通常比继续翻平台页更快。

## 1. 为什么客户端一直连不上

先按顺序检查这四件事：

1. 地址是不是客户端长连接地址
2. `uid` 和 `token` 是否由你的业务服务端正确生成
3. 当前地址、token、环境是不是同一套
4. 服务端 `health` 和路由接口是否正常

最常见的错误是把 `5001` 这类 HTTP API 地址当成客户端连接地址。

## 2. 为什么明明登录成功了，还是鉴权失败

登录你的业务系统成功，不等于 WuKongIM 鉴权一定成功。

客户端连接依赖的是：

- `uid`
- `token`
- 服务端对应的鉴权配置

只要三者有一项不匹配，就可能出现“业务登录没问题，但 IM 连不上”的情况。

## 3. 为什么能连接，但发不了消息

先区分是哪一层失败：

- 连接没问题，但发送被拒绝
- 消息发出去了，但目标端没收到
- 发送状态前端没有正确更新

通常优先检查：

- 目标频道是否正确
- `channel_id` / `channel_type` 是否正确
- 前端有没有监听发送状态
- 服务端有没有限制或鉴权问题

## 4. 为什么会话列表不更新

如果你走的是完整 SDK，很多人会误以为 SDK 会自动帮你补齐所有数据。

实际上你通常还需要自己实现：

- 会话同步数据源
- 消息同步数据源
- 频道资料数据源

没接这些回调时，连接成功不代表会话能力已经完整可用。

## 5. 为什么历史消息拉不到

最常见原因不是“服务端没有消息”，而是：

- 没实现 `syncMessagesCallback`
- 频道上下文传错
- 起止消息序号或拉取方向用错

先确认你传入的是正确频道，再确认当前 SDK 路径是否已经接好历史消息数据源。

## 6. 为什么多端状态看起来不一致

这通常和三个概念混淆有关：

- 用户
- 设备
- 会话

比如你以为清的是“所有端未读”，实际只更新了当前端会话状态；或者你以为是用户离线，实际只是某台设备掉线。

## 7. 快速接入和完整 SDK 应该怎么选

一句话判断：

- 先跑通链路，选 `快速接入 SDK`
- 要做完整消息、会话、频道能力，选 `完整 SDK`

如果你已经开始考虑历史消息、最近会话、频道资料这些问题，基本就该切到完整 SDK 了。
