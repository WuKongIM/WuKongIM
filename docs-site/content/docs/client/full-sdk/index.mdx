title: 完整 SDK
description: 面向需要完整 IM 能力和更深度定制能力的项目。
---

如果你只是想先连上服务器、收发一条消息，优先看“快速接入 SDK”。

如果你已经明确要把完整 IM 能力接进自己的应用，包括：

- 长连接管理
- 历史消息同步
- 会话列表维护
- 频道资料和成员能力
- 自定义消息与更深度业务集成

那就应该走“完整 SDK”这条路径。

## 完整 SDK 适合什么项目

- 已经有自己的用户体系和业务服务端
- 需要做聊天列表、历史消息、未读、群资料等完整功能
- 希望在客户端保留更多可控能力，而不是只做一个最短 Demo

## 建议怎么读

优先看你真正要接入的平台，再顺着“集成 -> 连接 -> 消息 -> 会话 -> 频道”的顺序往下读。
