iOS SDK 面向需要在原生 iPhone 或 iPad 应用里接入完整 IM 能力的项目。

如果你准备做的是：

- 原生聊天界面
- 会话列表和未读
- 历史消息同步
- 频道资料、成员、自定义业务消息

那就应该按 iOS 完整 SDK 的思路来理解接入，而不是只看最小 Demo。

如果你只是要先连通服务器、验证一条消息，优先看 [`快速接入 SDK`](/client/easy-sdk/ios)。  
如果你要做正式产品，再继续走当前这条路径。

## iOS 完整 SDK 更适合什么场景

- 原生 iOS IM、社交、客服、企业协同应用
- 对体验和状态控制要求高，不想只接一层轻封装
- 需要把消息同步、会话列表、未读、频道信息真正做进 App
- 已有业务后端，准备把登录态、鉴权、业务扩展一起整合

## 开始前先确认

### 1. Token 不是客户端硬编码出来的

真正的生产环境里，Token 应该由你的业务服务端签发，然后下发给 iOS 客户端用于连接 IM。

### 2. 长连接地址建议和部署配置一致

尤其在有测试环境、生产环境、公网和内网区分时，建议通过接口获取连接地址，而不是把地址散落在客户端代码里。

### 3. 历史消息和会话能力要一并设计

如果只看“如何发消息”，后面通常会补很多坑。  
完整 SDK 路径更适合先把同步和会话模型理解清楚。
