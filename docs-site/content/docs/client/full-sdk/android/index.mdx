Android SDK 适合需要在原生 Android 应用里接入完整 IM 能力的项目。

如果你的目标不只是“先收发一条消息”，而是准备把下面这些能力真正做进 App：

- 长连接生命周期管理
- 本地消息与历史同步
- 最近会话列表
- 频道资料与成员能力
- 自定义消息和业务扩展

那就应该走 Android 完整 SDK 路径。

如果你现在只是想先验证链路是否能通，优先看 [`快速接入 SDK`](/client/easy-sdk/android)。  
如果你已经确定要做正式聊天能力，再继续看这一组文档。

## Android 完整 SDK 更适合什么项目

- 原生 IM、社交、客服、协同类 Android App
- 需要自己控制连接、消息同步、会话和未读逻辑
- 已经有业务服务端，准备把 Token、地址发现、业务扩展接起来
- 不满足于 Demo 跑通，希望后续长期维护和演进

## 开始前先想清楚 3 件事

### 1. 登录态和 Token 由谁发

客户端最终连接的是 IM 长连接，但 Token 通常应该由你的业务服务端签发。  
所以 Android 接入前，先把服务端鉴权流程看清楚。

### 2. 连接地址是写死还是动态下发

如果你的环境有公网、内网、网关或多套部署，建议先理解连接地址获取方式，而不是把地址直接写死在 App 里。

### 3. 历史消息和会话列表怎么补齐

完整 SDK 不只是发消息。真正上线时，消息同步、最近会话、未读状态通常都要一起考虑。
