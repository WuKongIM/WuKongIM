---
title: 会话
description: 说明 Web SDK 的会话列表、未读与同步能力。
---

Web SDK 的会话能力主要围绕 `conversationManager` 展开。

它解决的不是“消息怎么收”，而是“聊天列表怎么维护”。

## 会话在客户端里主要负责什么

- 最近会话列表
- 未读数
- 会话排序
- 最后一条消息摘要
- 进入会话后的状态更新

如果你的页面里有“左侧聊天列表”或“最近联系人列表”，本质上就是会话能力。

## 1. 先实现会话同步数据源

完整 SDK 不会替你直接调用业务数据库，所以需要你挂自己的回调：

```ts
WKSDK.shared().config.provider.syncConversationsCallback = async () => {
  return fetchYourConversations();
};
```

这一步通常由你的业务服务端完成，再返回客户端真正要展示的会话信息。

## 2. 首次进入时同步最近会话

```ts
const conversations = await WKSDK.shared().conversationManager.sync({});
```

一般第一次进入应用或用户完成登录后做一次即可，后续更多依赖监听增量变化。

## 3. 监听会话变化

```ts
const onConversation = (conversation, action) => {
  console.log(action, conversation);
};

WKSDK.shared().conversationManager.addConversationListener(onConversation);
```

这一步通常用来更新：

- 新增会话
- 最后一条消息变化
- 未读数变化
- 会话被移除

## 4. 常见会话操作

这些是客户端里最常见的几个动作：

```ts
const current = WKSDK.shared().conversationManager.findConversation(channel);
const unread = WKSDK.shared().conversationManager.getAllUnreadCount();

WKSDK.shared().conversationManager.removeConversation(channel);
```

你可以把它们理解成“围绕会话列表做 UI 更新的基础能力”。

## 5. 一个高频场景怎么处理

典型流程通常是：

1. 用户登录后同步最近会话
2. 页面监听会话变化
3. 用户点击某个会话进入消息页
4. 清掉该会话未读
5. 新消息到达时更新会话排序和摘要

## 最常见的误区

- 把会话列表当成消息列表来渲染
- 每次新消息都重新全量拉一次会话
- 删除会话时，误以为底层频道和消息也会一起被物理删除
- 只做消息页，不做会话层，最后未读和排序逻辑越来越乱
