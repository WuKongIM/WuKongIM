title: 连接
description: 说明 Web SDK 的连接建立、重连和连接状态。
---

Web SDK 的连接能力主要围绕 `connectManager` 和全局 `config` 展开。

这页最重要的目标不是“把所有连接 API 列全”，而是帮你避免最常见的三类错误：

- 地址拿错
- 凭证没配对
- 连接状态有变化，但页面没有正确感知

## 1. 连接前至少要准备三项配置

```ts
import WKSDK from 'wukongimjssdk';

const config = WKSDK.shared().config;
config.uid = 'u10001';
config.token = 'your-token';
config.addr = 'ws://your-wukongim-server:5200';

WKSDK.shared().config = config;
```

这里最关键的是：

- `uid` 和 `token` 来自你的业务服务端
- `addr` 是客户端真正要连的 WebSocket 地址

## 2. 生产环境更推荐先拿路由地址，再连接

在 demo 代码里，Web 端会先请求路由接口，再从返回结果里选择连接地址：

```ts
const res = await fetchRoute(uid);
const addr = res.wss_addr || res.ws_addr;
```

这比把地址固定写死更适合生产环境，尤其是你已经有多节点、网关或对外域名配置的时候。

## 3. 建立连接

```ts
WKSDK.shared().connect();
```

对应断开连接：

```ts
WKSDK.shared().disconnect();
```

如果是页面卸载、退出登录或切账号，记得把监听和连接一起清理掉。

## 4. 监听连接状态

这是 Web 端最应该先补上的能力之一。

```ts
import { ConnectStatus } from 'wukongimjssdk';

const onConnectStatus = (status, reasonCode, connectionInfo) => {
  if (status === ConnectStatus.Connected) {
    console.log('connected', connectionInfo?.nodeId);
  } else {
    console.log('disconnected', reasonCode);
  }
};

WKSDK.shared().connectManager.addConnectStatusListener(onConnectStatus);
```

页面销毁时记得移除：

```ts
WKSDK.shared().connectManager.removeConnectStatusListener(onConnectStatus);
```

## 5. 连接状态通常要驱动什么 UI

实际项目里，连接状态不只是打印日志，它通常会直接影响：

- 顶部在线/离线状态展示
- 是否允许发送消息
- 是否提示重连中
- 是否触发重新同步会话和消息

所以这块不要只当成调试代码写一下就算了。

## 6. 一个最小可用连接流程

建议按这个顺序接：

1. 登录业务系统，拿到 `uid` 和 `token`
2. 通过路由接口或业务服务端拿到 `addr`
3. 设置 `WKSDK.shared().config`
4. 先注册连接状态监听
5. 再调用 `connect()`
6. 连接成功后再进入消息、会话和频道页面

## 最常见的错误

### 错误 1：把 `5001` 当成 WebSocket 地址

Web SDK 要连接的是长连接地址，不是管理或 HTTP API 地址。

### 错误 2：`uid`、`token`、`addr` 不是同一套环境

比如你把测试环境 token 和生产环境地址混在一起，表面看配置都填了，实际上一定连不上。

### 错误 3：只连接，不监听状态

这样一旦掉线、鉴权失败、地址配置错误，前端页面很难正确反馈。

### 错误 4：组件销毁后不清理监听

这会导致重复监听、重复渲染，后面排查会越来越乱。
