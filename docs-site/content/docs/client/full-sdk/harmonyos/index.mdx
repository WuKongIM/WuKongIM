HarmonyOS SDK 面向需要在鸿蒙生态里接入完整实时消息能力的项目。

如果你需要的不只是一个演示连接，而是完整的消息、会话、频道和业务扩展能力，这条路径会比“快速接入”更合适。

HarmonyOS 当前没有单独的“快速接入”入口页，因此这里更适合作为鸿蒙平台接入的主入口。

## HarmonyOS 完整 SDK 适合什么项目

- 面向鸿蒙终端的正式 IM 应用
- 希望消息、会话、频道和业务能力一起接入
- 已经确认要走长期维护的正式方案，而不是临时演示
- 需要和已有业务用户体系、鉴权服务、业务接口打通

## 接入前先把这 3 个问题想清楚

### 1. 用户怎么登录，Token 从哪里来

HarmonyOS 客户端连接 IM 之前，仍然需要你的业务服务端负责登录态和 Token 签发。

### 2. 客户端连接哪个地址

如果有不同环境或网络入口，建议先理解地址发现接口，再决定是固定地址还是动态下发。

### 3. 本地同步要做到什么程度

完整 SDK 通常意味着你不仅要连上服务器，还要考虑：

- 历史消息补拉
- 会话列表维护
- 未读和状态同步
