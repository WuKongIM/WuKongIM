如果你已经把服务跑起来，Web 是最适合先验证接入链路的平台。这个页面只保留最短路径。

## 第一步：安装 SDK

```bash
npm install easyjssdk
```

也可以用 CDN，但如果你准备继续开发，优先推荐 `npm`。

## 第二步：初始化

```ts
import { WKIM, WKIMChannelType, WKIMEvent } from 'easyjssdk';

const im = WKIM.init('ws://your-wukongim-server.com:5200', {
  uid: 'your_user_id',
  token: 'your_auth_token',
});
```

这里最重要的是两点：

- 连接地址是 `5200` 的 WebSocket 地址，不是 `5001`
- `uid` 和 `token` 应由你的业务服务端准备

## 第三步：监听连接和消息

```ts
im.on(WKIMEvent.Connect, async () => {
  console.log('IM connected');

  await im.send(
    'target-user-id',
    WKIMChannelType.Person,
    { type: 'text', content: 'Hello from EasyJSSDK!' },
  );
});

im.on(WKIMEvent.Message, (message) => {
  console.log('Received:', message);
});

im.on(WKIMEvent.Error, (error) => {
  console.error('IM error:', error);
});
```

## 第四步：建立连接

```ts
await im.connect();
```

## 成功的判断标准

- `Connect` 事件触发
- 发送后另一端能收到消息
- 没有持续重连或鉴权错误

## 最常见的错误

- 把 `5001` 当成客户端连接地址
- 服务端没有正确准备 `uid` 和 `token`
- `WK_EXTERNAL_IP` 或 WebSocket 对外地址配置错误
