`快速接入 SDK` 只解决一件事：

**用最短路径先把连接和收发消息链路跑通。**

如果你当前的目标是下面这些之一，优先选这条路径：

- 先做一个 Demo
- 先验证服务端鉴权和连接链路
- 项目时间紧，先证明能接通
- 当前不打算立刻做完整会话、历史消息和频道管理

## 这条路径适合什么，不适合什么

### 适合

- 原型验证
- MVP
- 演示环境
- 首次接入测试

### 不适合

- 要做完整聊天列表和历史消息
- 要深度自定义消息、频道、成员和本地数据层
- 已经明确要长期维护完整 IM 客户端

如果你已经进入“不只是连上，还要把完整能力接起来”的阶段，应该切到 [`完整 SDK`](/client/full-sdk)。

## 快速接入通常只做四步

1. 安装 SDK
2. 准备 `uid`、`token` 和 WebSocket 地址
3. 监听连接和消息事件
4. 发起连接并验证收发

## 开始前先确认两件事

### 1. 你拿到的是客户端连接地址

- Web 客户端通常走 `5200`
- 不要把管理或 HTTP API 地址当成长连接地址

### 2. `uid` 和 `token` 由你的业务服务端准备

客户端接入成功与否，很多时候不是 SDK 问题，而是服务端的鉴权和路由链路没准备好。

## 跑通后的下一步

如果快速接入已经成功，通常下一步只需要在两条路里二选一：

- 继续补业务 UI 和基础消息体验
- 切到 [`完整 SDK`](/client/full-sdk) 做会话、历史消息、频道等完整能力
