WuKongIM 现在可以先按两条路径理解：

- `快速接入 SDK`：尽快接起来，先把功能跑通
- `完整 SDK`：需要完整 IM 能力和更深度定制

先决定这件事，再进入具体平台页面，会比一上来就钻平台文档更省时间。

## 先给结论

### 选快速接入 SDK，如果你是下面这些情况

- 想在几分钟内把连接和基本消息跑通
- 做 MVP、原型验证或演示环境
- 只需要核心聊天能力，不打算一开始就做深度定制
- 团队希望先验证产品，再逐步升级方案

### 选完整 SDK，如果你是下面这些情况

- 要做企业级或长期演进的 IM 产品
- 需要完整会话、频道、数据源、高级能力
- 对 UI、业务流程、性能细节有更深定制要求
- 你已经明确把 WuKongIM 当成长期基础设施，而不是一次性接入

## 快速对比

| 对比项 | 快速接入 SDK | 完整 SDK |
| --- | --- | --- |
| 集成时间 | 很短，适合尽快跑通 | 更长，需要认真接入 |
| 学习成本 | 低 | 中等 |
| 功能完整性 | 先覆盖核心能力 | 完整能力更全 |
| 自定义空间 | 有限 | 更高 |
| 典型场景 | MVP、演示、快速验证 | 企业级、复杂业务、长期维护 |

## 平台支持

### 快速接入 SDK

- Web
- iOS
- Android
- Flutter

### 完整 SDK

- Web
- iOS
- Android
- Flutter
- HarmonyOS
- 其他平台扩展

## 推荐选择方法

1. 如果你还没跑通过最小链路，先走 `快速接入 SDK`
2. 如果你已经确定要深度定制聊天能力，直接看 `完整 SDK`
3. 如果你不确定，先用快速接入验证业务，再迁到完整 SDK
