第一次体验 WuKongIM，不需要先准备完整生产环境，但至少要确认下面这些条件。

## 最小要求

- 一台 Linux 机器，或可运行 Linux 容器的环境
- Docker 24 或更高版本
- 可开放 `5001`、`5100`、`5200`、`5300`、`5172`、`11110`

## 推荐资源

| 场景 | 推荐配置 |
| --- | --- |
| 本地学习 / 演示 | 2 核 4G |
| 稳定体验 / 小规模验证 | 4 核 8G |
| 准备继续扩展 | 4 核 16G 或更高 |

## 先理解几个端口

- `5001`：HTTP API
- `5100`：TCP 长连接
- `5200`：WebSocket 长连接
- `5300`：管理后台
- `5172`：聊天 Demo
- `11110`：集群节点通信

第一次体验时，最重要的是确认 `5001`、`5200`、`5300`、`5172` 可访问。

## 启动前检查清单

- 你知道当前机器的外网 IP 和内网 IP
- Docker 可以正常拉取镜像并启动容器
- `5001` 不会直接暴露给公网业务用户
- 你已经决定先走 Docker 单机，而不是一开始就做集群
