---
title: 连接测试
description: 通过最小验证步骤确认服务已启动、端口可访问、连接链路可用。
---

这一步只做最小验证，不讨论复杂业务逻辑。目标是确认服务活着、地址对、Demo 能进。

## 1. 检查健康状态

```bash
curl http://127.0.0.1:5001/health
```

预期返回：

```json
{"status":"ok"}
```

如果这里不通，先不要继续看 SDK 或 API。

## 2. 验证管理后台

浏览器打开：

```text
http://127.0.0.1:5300/web
```

如果页面能打开，说明管理侧服务已经起来。

## 3. 验证聊天 Demo

浏览器打开：

```text
http://127.0.0.1:5172/chatdemo
```

随便用两个用户登录，互发一条消息。如果消息能实时到达，说明最小链路已经通了。

## 4. 验证客户端连接地址

如果你准备自己接 SDK，再确认下面两个地址：

- TCP：`<host>:5100`
- WebSocket：`ws://<host>:5200`

## 常见失败点

- 容器起来了，但 `5001` 或 `5200` 没开放
- `WK_EXTERNAL_IP` 配成了错误地址
- 机器上还有防火墙或安全组没放行
- 使用公网地址时没有正确区分公网和内网配置
