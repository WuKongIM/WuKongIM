WuKongIM 的核心能力可以先归纳成五件事：简单、性能、可靠、可扩展、接入完整。

如果你现在还在判断“它到底强在哪”，这一页就是为了让你先抓重点，而不是先陷进接口或部署细节。

## 1. 从单机起步足够简单

- 零依赖起步，不要求先准备额外中间件
- 支持 Docker 直接体验，也支持用本地配置跑单机
- 文档路径按“先跑起来，再接入”组织，适合第一次评估

## 2. 面向实时通信的性能设计

- 自定义二进制协议，尽量降低带宽和客户端耗电
- 针对即时通信负载优化消息传输和存储路径
- 能承载大规模在线连接与高频消息流

## 3. 消息可靠性是内建能力

- 消息支持持久化存储，而不是只做临时投递
- 会话、离线消息、同步链路可以作为完整能力使用
- 节点异常时系统仍可以继续运行和恢复

## 4. 可以往集群演进

- 单机部署适合早期项目或本地验证
- 进入生产后可以切到多节点集群
- 扩容、升级、迁移都有对应的运维路径

## 5. 客户端和服务端路径都完整

- 客户端侧有快速接入 SDK 和完整 SDK 两条路径
- 服务端侧提供 REST API、Webhook、管理与系统接口
- 文档按客户端接入、服务端集成、部署运维拆开，角色边界清晰

## 看能力时最容易忽略的点

- “简单”不是只能做 Demo，而是可以从单机平滑起步
- “高性能”不只是高并发数字，还包括协议和存储路径设计
- “可靠”不只是消息不丢，还包括同步、会话和恢复链路
- “完整接入”不只是 SDK 多，而是客户端、服务端、运维路径都完整
