设备表示“同一个用户从哪个终端接入 WuKongIM”。

如果用户是身份主体，那么设备就是这个身份在不同终端上的具体落点。

## 为什么设备概念很重要

因为即时通信系统很少只有一个终端。

同一个用户可能同时登录在：

- Web
- iPhone
- Android
- iPad
- 桌面端

这时候系统要处理的就不只是“用户在不在线”，还包括：

- 哪些设备在线
- 哪些设备掉线
- 某条消息要不要同步到其他设备
- 是否需要强制某个设备退出

## 设备和连接的关系

- 设备是终端身份
- 连接是设备当前建立的网络通道

通常可以粗略理解成：一个在线设备会有一条当前活跃连接。但在排障时，真正变化更频繁的是连接，不是设备标识。

## 你什么时候会关心设备

### 1. 多端登录

同一用户在多个终端同时在线时，需要明确哪些行为是“同步给其他设备”，哪些不是。

### 2. 在线状态

很多业务关心的是“这个用户是否在线”，但底层往往先来自设备在线状态汇总。

### 3. 强制下线

比如账号安全、风控、后台管理、切换账号时，可能需要让某个设备退出。

## 设备相关能力一般放在哪里

- 客户端 SDK 负责上报和维持连接
- 服务端接口负责查询在线状态、强制设备退出
- 业务系统决定一台设备是否允许继续登录

## 最容易混淆的两个点

### 误区 1：用户在线就代表所有设备都在线

不一定。可能只是其中一台设备在线。

### 误区 2：强制退出是“踢用户”，不是“踢设备”

实际要看你调用的是面向用户还是面向设备的控制能力，业务上要先明确范围。
