---
title: 升级
description: Kubernetes 环境下 WuKongIM 的升级建议。
---

Kubernetes 下升级 WuKongIM，表面上是一次 `helm upgrade`，但真正要控制的是两类风险：

- 服务版本变更风险
- Chart / values 变更风险

## 升级前先确认

### 1. 看发布说明

先确认目标版本有没有这些变化：

- 配置项调整
- 存储或集群行为变化
- API / Webhook / SDK 兼容性说明

### 2. 看 Helm 参数是否有变化

升级前最好保留当前使用的 values，避免升级时因为参数差异把部署形态顺手改掉。

### 3. 先决定是否需要灰度或预演

如果是生产环境，建议先在测试或预发布环境验证：

- Pod 是否正常滚动
- 客户端能否稳定连接
- 消息和会话同步是否正常

## 基本升级步骤

```bash
helm repo update
helm search repo wukongim
```

然后执行升级：

```bash
helm upgrade wkim wukongim/wukongim \
  -n wukongim \
  --version <chart-version>
```

如果你平时使用自定义 values，也要一并带上。

## 升级后至少检查什么

```bash
kubectl get pods -n wukongim
kubectl get svc -n wukongim
helm status wkim -n wukongim
```

然后再做业务验证：

1. `health` 正常
2. 客户端连接正常
3. 单发消息正常
4. 会话和历史消息同步正常

## 最常见的坑

- 只看 Helm 命令执行成功，不看滚动过程是否有异常
- 忘了带原本的 values
- 没看发布说明，直接跨版本升级
- 升级完成后只看 Pod 状态，不做消息链路验证
