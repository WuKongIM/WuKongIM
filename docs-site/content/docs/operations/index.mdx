`部署与运维` 负责回答的是：

- 我现在该选哪种部署方式
- 什么时候该从单机切到集群
- 配置、监控、扩容、升级、排障分别去哪看

如果你还没把 WuKongIM 跑起来，不要先在这一栏里深挖，优先回 [`快速开始`](/quick-start)。

## 最常见的 4 条路径

### 1. 还没部署，先决定怎么上

先看：

- [`部署方案选择`](/operations/deployment-selection)
- [`单机还是集群`](/operations/single-node-vs-cluster)

### 2. 已经跑起来，开始管配置

先看：

- [`配置说明`](/operations/configuration)
- [`可观测性`](/operations/observability)

### 3. 要扩容、升级或迁移

先看：

- 扩容：各部署方式下的 `scaling`
- 升级：各部署方式下的 `upgrade`
- 迁移：[`数据迁移`](/operations/data-migration)

### 4. 出问题了，不知道从哪查

先看：

- [`故障排查`](/operations/troubleshooting)
- [`可观测性`](/operations/observability)

## 如果你不知道该从哪一页开始

经验上最省时间的顺序是：

1. 先决定单机还是集群
2. 再决定 Docker、Linux 还是 Kubernetes
3. 跑通后再进入配置、观测和排障
