---
title: 单机部署
description: 不依赖 Docker，直接用二进制在 Linux 上先跑通一个单机节点。
---

Linux 单机部署适合这几类场景：

- 想最小化运行依赖
- 希望自己接管进程、配置和目录结构
- 准备先用单机验证，再扩到集群

如果你只是第一次体验 WuKongIM，优先还是建议先走 [`Docker 单机部署`](/quick-start/docker-single-node)。  
如果你明确要在 Linux 机器上直接跑二进制，再继续看这页。

## 适用场景

- 开发、测试环境
- 小型业务先上线
- 自己维护进程和配置文件的传统服务器环境

单机的优点是简单直接、性能不错；缺点是没有容灾，需要自己做好备份。

## 环境建议

- Linux 系统
- 建议 2 核 4G 起步，生产更建议 4 核 8G 或更高

## 第一步：准备二进制

你有两种做法：

- 从 GitHub Releases 下载对应架构的 `wukongim` 可执行文件
- 或者在源码仓库里自己构建

如果你在当前仓库里自己构建：

```bash
go build -o wukongim main.go
```

然后赋予执行权限：

```bash
chmod +x wukongim
```

## 第二步：准备配置文件

可以直接从仓库示例开始：

```text
exampleconfig/single.yaml
```

如果你想先写一个最小版本，至少要把下面这些值配对：

```yaml
mode: "debug"
addr: "tcp://0.0.0.0:5100"
httpAddr: "0.0.0.0:5001"
wsAddr: "ws://0.0.0.0:5200"
rootDir: "./wukongimdata/1"
intranet:
  tcpAddr: "127.0.0.1:5100"
manager:
  on: true
  addr: "0.0.0.0:5300"
demo:
  on: true
cluster:
  nodeId: 1
  addr: "tcp://127.0.0.1:10001"
  serverAddr: "127.0.0.1:10001"
  apiUrl: "http://127.0.0.1:5001"
auth:
  superToken: "replace-with-your-token"
  kind: "jwt"
jwt:
  secret: "replace-with-your-secret"
  expire: 30d
```

## 第三步：重点确认这几类配置

### 监听地址

- `addr`
- `httpAddr`
- `wsAddr`

这是服务自己绑定的地址。

### 数据目录

- `rootDir`

这个目录需要提前规划好磁盘空间和备份策略。

### 集群相关字段

即使是单机，也会看到：

- `cluster.nodeId`
- `cluster.addr`
- `cluster.serverAddr`
- `cluster.apiUrl`

单机场景下，它们可以都指向本机。

### 认证相关

当前仓库示例已经包含：

- `auth.superToken`
- `auth.kind`
- `jwt.secret`

所以不要把服务端接口简单理解成“默认完全无鉴权”。

## 第四步：启动服务

```bash
./wukongim --config ./exampleconfig/single.yaml
```

如果你用的是自己写的配置文件，就把路径替换成你的文件。

停止服务：

```bash
./wukongim stop
```

## 常用端口

| 端口 | 用途 |
| --- | --- |
| `5001` | HTTP API |
| `5100` | TCP |
| `5200` | WebSocket |
| `5300` | 管理后台 |
| `5172` | Demo |

## 怎么验证是否成功

先看健康检查：

```bash
curl http://127.0.0.1:5001/health
```

再访问：

- 管理后台：`http://127.0.0.1:5300/web`
- Demo：`http://127.0.0.1:5172/chatdemo`

如果管理后台可打开，Demo 可以登录并收发消息，说明单机链路已经跑通。

## 常见坑

### 1. 监听地址和对外地址混了

监听地址是服务绑定的地址，对外地址是客户端真正访问的地址。

单机测试时可以先不复杂化，但正式可访问场景一定要分清。

### 2. 用单机配置，却把 `serverAddr` 写成不可达地址

即使是单机，这个地址也要保证本机可用。

### 3. 忘了处理认证配置

如果你后面准备让业务系统调服务端接口，最好尽早把 `superToken` 和 JWT 配好，而不是上线前再补。
