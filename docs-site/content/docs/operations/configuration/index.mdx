配置页最容易让新手一上来就看晕，因为 `wk.yaml` 注释很多、参数也不少。

这一组文档先帮你建立一个清晰判断：

- 哪些是单机和集群都会用到的基础配置
- 哪些是进入集群后才需要重点理解的配置

## WuKongIM 主要有两种配置方式

- YAML 配置文件
- 环境变量

默认建议先读 YAML，再在部署环境里按需用环境变量覆盖。

一个关键规则是：

**环境变量优先级高于配置文件。**

所以如果你发现“明明改了 yaml 但没生效”，先查是不是部署环境里已经有同名环境变量覆盖了。

## 先按你当前的问题进入

### 基础配置

适合先解决这些问题：

- 服务监听在哪
- 客户端最终连到哪
- 数据存到哪
- 管理后台和 Demo 怎么开
- 鉴权开不开、怎么配

入口页：[`基础配置`](/operations/configuration/basic)

### 集群配置

适合已经进入多节点场景后再重点理解：

- 每个节点是谁
- 节点之间如何互通
- 初始集群如何组织
- 后续节点如何加入

入口页：[`集群配置`](/operations/configuration/cluster)

## 当前最值得对照的配置文件

- `config/wk.yaml`
  适合先理解有哪些配置块
- `exampleconfig/single.yaml`
  适合先对照单机最小可运行配置
- `exampleconfig/cluster1.yaml`
  适合先对照集群节点样例

## 配置时最容易混淆的三件事

### 1. 监听地址和对外地址不是一回事

服务自己监听在哪，和客户端最终应该连到哪，往往不是同一个值。

### 2. 单机能跑，不代表集群参数就完整

集群里必须额外关心节点间互访，不只是本机端口监听。

### 3. 改了配置，不代表实际生效

因为可能存在：

- 环境变量覆盖
- 旧进程未重启
- 网关入口未同步更新

## 最省时间的配置顺序

1. 先把监听地址、对外地址、数据目录配对
2. 再确认鉴权、管理端、Demo 这类开关
3. 进入集群后再补节点和互联配置
4. 改完配置后一定重启并验证实际生效
