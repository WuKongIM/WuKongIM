这页负责回答一个很实际的问题：

**我不想先通读文档，能不能先看现成效果和样例代码？**

可以。WuKongIM 当前最值得先看的入口主要分三类：

- 在线 Demo
- 仓库内置演示入口
- 各平台 SDK 示例工程

## 先给结论

如果你现在的目标是“先看到效果，再决定怎么接”，推荐顺序是：

1. 先看在线 Demo，确认产品形态
2. 再跑本地 Demo，确认你自己的链路是否打通
3. 最后进入对应 SDK 文档，开始正式接入

## 1. 在线 Demo

当前公开演示入口包括：

- 聊天 Demo：`http://imdemo.githubim.com`
- 监控 Demo：`http://monitor.githubim.com/web`

如果你只是想快速感受产品形态，这两个入口最直接。

## 2. 仓库本地 Demo

如果你已经把本地或测试环境跑起来，通常可以直接访问：

- 单机默认：`http://127.0.0.1:5172/chatdemo`
- Docker 集群示例：`http://127.0.0.1:15172/chatdemo`

这比读文档更适合先确认“整个链路是不是已经通了”。

## 3. SDK 示例仓库

如果你已经确定要接哪个平台，下一步就应该看对应 SDK 示例工程。常见入口包括：

- `WuKongIMAndroidSDK`
- `WuKongIMiOSSDK`
- `WuKongIMJSSDK`
- `WuKongIMFlutterSDK`
- `WuKongIMHarmonyOSSDK`

每个仓库通常都会附带自己的 example 工程。

## 推荐怎么用这些示例

### 先看效果，再回文档

最省时间的顺序通常是：

1. 先打开在线 Demo 或本地 Demo
2. 再看对应 SDK 选择页
3. 最后才去看具体 API 或 SDK 细节

### 不要把 Demo 当成生产接入模板

Demo 最适合：

- 理解最小链路
- 验证地址、Token、连接是否正常

但正式接入时，还是要回到：

- 你的鉴权逻辑
- 你的业务消息模型
- 你的部署方式

## 不同阶段最适合看什么

- 还没决定选哪种 SDK：看 [`SDK 选型`](/client/sdk-selection)
- 还没跑通服务端：看 [`快速开始`](/quick-start)
- 想先理解平台接入路径：看 [`客户端接入`](/client)
- 想找更多生态入口：看 [`生态与仓库`](/resources/ecosystem)
