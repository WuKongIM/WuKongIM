---
title: 批量发送
description: 把同一条消息一次发送给多个用户时，优先使用这个接口。
---

核心接口：

```text
POST /message/sendbatch
```

这个接口的定位很明确：

- 一条相同内容
- 发给多个用户
- 目标是个人频道

如果你要给不同频道发送不同内容，不要用它，改用 [`发送消息`](/server/message/send) 逐条发。

## 什么时候适合用

- 系统通知
- 运营广播
- 给一批用户推送相同提醒

它的优势是请求体更简单，不需要你为每个用户分别组织一条消息。

## 请求体长什么样

和旧文档里“消息数组”不同，当前实现接收的是**单个对象**，不是数组。

| 参数 | 必填 | 说明 |
| --- | --- | --- |
| `from_uid` | 是 | 发送者 UID |
| `payload` | 是 | Base64 编码后的消息内容 |
| `subscribers` | 是 | 接收这条消息的用户 UID 列表 |
| `header.no_persist` | 否 | 是否不持久化 |
| `header.red_dot` | 否 | 是否显示红点 |
| `header.sync_once` | 否 | 是否只同步一次 |

## 最小示例

```bash
curl -X POST "http://localhost:5001/message/sendbatch" \
  -H "Content-Type: application/json" \
  -d '{
    "from_uid": "system",
    "payload": "57O757uf5YWs5ZGK77ya5LuK5pmaIDIyOjAwIOezu+e7n+e7tOaKpOOAgg==",
    "subscribers": ["u1001", "u1002", "u1003"],
    "header": {
      "red_dot": 1
    }
  }'
```

## 返回什么

这个接口不会像单条发送那样返回 `message_id`。

它返回的是这次批量发送里失败的用户列表：

```json
{
  "fail_uids": [],
  "reason": []
}
```

怎么理解：

- `fail_uids` 为空：这批都发成功了
- `fail_uids` 不为空：表示部分用户发送失败
- `reason`：和失败用户一一对应的错误原因

## 和单条发送的区别

| 场景 | 用哪个接口 |
| --- | --- |
| 给一个频道发一条消息 | `/message/send` |
| 给多个用户发同一条消息 | `/message/sendbatch` |
| 给多个频道分别发不同消息 | 多次调用 `/message/send` |

## 常见坑

### 1. 这不是“多条消息批量提交”

当前仓库实现里，`/message/sendbatch` 表示：

**同一条消息，批量投递给多个订阅者。**

不是把多条消息对象打成数组一起提交。

### 2. `subscribers` 不能为空

这个接口本质上就是给一组 UID 批量投递，所以没有 `subscribers` 会直接报错。

### 3. 目标默认是个人频道

服务端会把 `subscribers` 中的每个 UID 当成个人频道目标处理。

如果你要发群消息，还是走普通发送接口。

### 4. `payload` 仍然要先编码

请求里虽然是 JSON，但 `payload` 依然是消息体字节内容，传 JSON 时会表现为 Base64 字符串。
