`消息` 这一组接口可以先按三个动作理解：

- **发送**：把消息写进某个频道
- **同步**：让客户端按序拉取历史消息
- **查询**：按条件搜索或定位消息

## 最常用的三个接口

| 目标 | 接口 | 说明 |
| --- | --- | --- |
| 发一条消息 | `POST /message/send` | 单条发送，最常用 |
| 批量发消息 | `POST /message/sendbatch` | 群发通知或批量推送 |
| 同步历史消息 | `POST /channel/messagesync` | 拉取频道历史消息 |

## 接口使用前先确认的事

### 1. `payload` 不是原始文本

消息内容通常要按协议组织后，再用 Base64 编码传输。

### 2. `channel_id` 直接用原始值

不要给 `channel_id` 加 `@` 前缀，直接传频道原始 ID。

### 3. 频道类型要明确

- `1`：个人频道
- `2`：群组频道
