用户接口主要解决三类问题：

- 给客户端准备和更新连接凭据
- 查询用户在线状态
- 管理用户设备连接

## 这些接口一般怎么用

### 登录或刷新凭据

先用 `POST /user/token` 写入或刷新某个用户在某个设备上的 token。

### 判断用户当前是否在线

用 `POST /user/onlinestatus` 一次查多个用户的在线状态。

### 处理异常连接或主动踢下线

用 `POST /user/device_quit` 强制指定设备退出。

## 阅读顺序建议

如果你正在接入登录流程，先看 [`更新用户 Token`](/server/user/token)。
