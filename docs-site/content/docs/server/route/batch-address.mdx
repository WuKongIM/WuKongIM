---
title: 批量获取地址
description: 一次拿到多个用户的连接地址，适合接入层或业务服务批量预取路由。
---

核心接口：

```text
POST /route/batch
```

这个接口和 [`获取地址`](/server/route/get-address) 的区别是：

- `/route` 处理“当前该连哪儿”
- `/route/batch` 处理“一批用户该连哪儿”

通常它更适合业务服务端、接入网关、批处理程序调用，而不是每个客户端自己频繁直调。

## 什么时候用

- 业务服务预先获取一批用户的连接地址
- 网关层为多用户分发连接目标
- 你需要把同一批用户按节点地址分组处理

## 查询参数

`intranet`

- `0`：返回对外连接地址
- `1`：返回内网连接地址

默认更常见的是 `0`。

## 请求体格式

这里有一个很容易写错的点：

**请求体是字符串数组，不是对象。**

也就是直接传 UID 列表：

```json
["u1001", "u1002", "u1003"]
```

## 最小示例

```bash
curl -X POST "http://localhost:5001/route/batch?intranet=0" \
  -H "Content-Type: application/json" \
  -d '["u1001", "u1002", "u1003"]'
```

## 返回什么

返回值是一个数组，每个元素表示“一组用户对应同一个连接地址”：

```json
[
  {
    "uids": ["u1001", "u1002", "u1003"],
    "tcp_addr": "127.0.0.1:5100",
    "ws_addr": "ws://127.0.0.1:5200",
    "wss_addr": ""
  }
]
```

## 怎么理解返回结构

| 字段 | 说明 |
| --- | --- |
| `uids` | 这一组用户 |
| `tcp_addr` | TCP 客户端连接地址 |
| `ws_addr` | WebSocket 连接地址 |
| `wss_addr` | TLS WebSocket 地址 |

即使当前环境只返回一个分组，也不要在调用方里写死“结果永远只有一条”。

从接口设计上，它允许未来把不同用户分配到不同地址组。

## 常见坑

### 1. 请求体写成 `{ "uids": [...] }`

当前实现直接绑定字符串数组，写成对象会报格式错误。

### 2. 把 API 地址当成客户端连接地址

`5001` 是 HTTP API，不是客户端长连接地址。

客户端真正要用的是返回里的：

- `tcp_addr`
- `ws_addr`
- `wss_addr`

### 3. 对外地址没配好

如果 `external.tcpAddr`、`external.wsAddr`、`external.wssAddr` 配置不正确，返回出来的地址也会跟着错。
