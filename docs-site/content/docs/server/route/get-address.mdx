---
title: 获取地址
description: 获取当前部署对外或内网连接地址的接口。
---

核心接口：

```text
GET /route
```

这个接口的作用不是鉴权，也不是发送消息，而是告诉客户端应该连哪个地址。

当前实现里，它返回的是“这套部署对外暴露的连接地址”，不是“按用户分配到某个不同节点”的动态路由接口。

## 什么时候用它

- 你不想在客户端里写死 TCP 或 WebSocket 地址
- 部署里有公网 / 内网地址之分
- 集群或网关场景下，需要统一下发连接地址

## 查询参数

这个接口当前只读取一个参数：`intranet`

- `0`：返回对外地址
- `1`：返回内网地址

要注意两点：

- 当前实现不会读取 `uid`
- 它也不会根据不同用户返回不同节点地址

最常用示例：

```bash
curl "http://localhost:5001/route?intranet=0"
```

## 返回内容

```json
{
  "tcp_addr": "127.0.0.1:5100",
  "ws_addr": "ws://127.0.0.1:5200",
  "wss_addr": "wss://127.0.0.1:5210"
}
```

如果 `intranet=1`，当前实现主要返回内网 `tcp_addr`；`ws_addr` 和 `wss_addr` 通常为空。

## 怎么理解这三个地址

- `tcp_addr`：App 或 TCP 客户端使用
- `ws_addr`：Web 客户端使用
- `wss_addr`：需要 TLS 的 Web 客户端使用

## 它和“用户路由”不是一回事

很多人第一次看到 `/route`，会自然理解成：

- 传一个用户 UID
- 服务端返回这个用户当前所在节点

但当前版本不是这个语义。  
它本质上更接近“获取连接入口地址”，适合作为客户端初始化连接前的第一步。

## 常见坑

- 把 `5001` 当成客户端连接地址
- 误以为 `/route` 会按用户返回不同节点
- 没有正确区分公网和内网地址
- 对外地址没有和 `external` 配置对应起来
