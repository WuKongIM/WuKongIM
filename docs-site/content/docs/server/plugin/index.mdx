`WuKongIM` 定义了一套插件规则，第三方可以实现这套规则，用于扩展或增强现有 `WuKongIM` 的消息处理逻辑。

比如可以通过插件实现：`敏感词过滤`、`消息搜索`、`AI` 等功能。

## 什么是插件

插件本质上是运行在 WuKongIM 外围、但能参与消息处理链路的独立程序。

如果你希望在不改动主服务核心逻辑的前提下扩展能力，插件是更合适的入口。

常见场景包括：

- 在消息发送前做敏感词过滤
- 在消息持久化后建立搜索索引
- 给指定用户挂接 AI 助手或机器人能力

## 用户插件

用户插件可以收到插件绑定用户的所有消息，安装后需要先绑定用户才能生效。

例如要实现与大模型聊天，可以把插件绑定到指定用户 UID，然后给这个用户发消息时，消息就会转给插件处理。

实现 `Receive` 函数的通常就是用户插件。

这类插件更适合：

- AI 助手
- 个人机器人
- 针对特定用户的定制化能力

## 全局插件

全局插件可以收到系统内的所有消息，安装后会对全局生效。

例如敏感词过滤插件、消息搜索插件，都属于全局插件，会监听每条发送消息或消息持久化事件。

这类插件更适合：

- 敏感词过滤
- 消息搜索
- 数据分析
- 统一审计或风控
