title: 集成到自己的系统

---

> 所有 WuKongIM 的接口都不应直接暴露给 App 端调用。推荐由前端先访问你自己的业务 API，再由业务 API 调用 WuKongIM，这样更安全，也更容易统一权限和审计。

## 第一步：对接业务系统用户

先把你自己的用户体系和 WuKongIM 的 `uid`、`token` 对齐。

![集成用户流程](/images/advance/integration.png)

推荐流程如下：

1. 你的应用先调用自己的登录或注册接口。
2. 业务接口处理完自身逻辑后，生成用户 `uid` 和 `token`。
3. 业务服务再调用 WuKongIM 的 [`更新用户 Token`](/server/user/token) 接口，把这些凭据同步到 WuKongIM。
4. 业务接口把 `uid` 和 `token` 返回给客户端。
5. 客户端再调用 WuKongIM SDK 的 `connect` 方法建立长连接。

只要客户端传入的 `uid`、`token` 与服务端同步给 WuKongIM 的一致，就可以完成连接校验。

## 第二步：提供 Webhook 接口

业务系统需要提供一个 HTTP API 给 WuKongIM 回调。

WuKongIM 会按 [`Webhook`](/server/webhook) 的约定，把用户状态变化、消息事件等内容以事件机制推送给你的业务系统。

这一步的目标不是让业务系统主动轮询 WuKongIM，而是让 WuKongIM 把关键事件主动通知给你。

## 第三步：配置 Webhook 到 WuKongIM

把你的回调地址配置到 WuKongIM：

```yaml
webhook:
  httpAddr: "http://xxxxx" # 通过此地址把事件通知给第三方业务系统
```

配置完成后，集成了 WuKongIM SDK 的应用产生的事件就会推送到你的业务系统里。

例如用户上线后：

1. 客户端 SDK 会把上线事件通知给 WuKongIM。
2. WuKongIM 再通过配置好的 Webhook 地址，把该事件推给你的业务服务。

## 集成时最重要的原则

- 客户端不要直连业务敏感接口和 WuKongIM 管理接口
- `uid` 一定使用你的业务主键，不要临时生成
- 事件回调入口要稳定，避免超时和频繁失败
- 用户鉴权、业务权限、消息事件处理要放在你自己的服务端收口
