> WuKongIM 压测功能只有大于或等于 `v2.1.2-20250120` 的版本才支持。

## 服务器开启压力配置

WuKongIM 服务端需要先打开压测相关配置：

<CodeGroup>

```yaml wk.yaml
...
stress: true
intranet:
  tcpAddr: "ip:port"
...
```

```bash environment
WK_STRESS=true
WK_INTRANET_TCPADDR=ip:port
```

</CodeGroup>

### 配置说明

`stress`

用于开启压测能力。只有开启后，服务器才能被压测机接入。

`intranet.tcpAddr`

- 单机模式且没有负载均衡时，填 WuKongIM 服务器的内网 IP 加 `5100` 端口，例如 `192.168.1.12:5100`
- 分布式并配置负载均衡时，填负载均衡内网 IP 加 `15100` 端口，例如 `192.168.1.11:15100`

## 安装压测机

压测机建议使用内存较大的服务器，推荐至少 `4 核 16G`。

### 下载执行文件

<CodeGroup>

```bash amd64
sudo curl -L -o wkstress https://github.com/WuKongIM/StressTester/releases/download/v1.1.0/wkstress-linux-amd64
```

```bash arm64
sudo curl -L -o wkstress https://github.com/WuKongIM/StressTester/releases/download/v1.1.0/wkstress-linux-arm64
```

</CodeGroup>

### 修改执行权限

```bash
sudo chmod +x wkstress
```

### 启动与停止

启动：

```bash
nohup ./wkstress &
```

默认端口是 `9466`。

停止：

```bash
kill -9 $(lsof -t -i :9466)
```

## 开始压测

### 1. 添加压测机器

在后台添加压测机，地址格式为 `http://ip:9466`，其中 `ip` 为压测机内网 IP。

### 2. 设置压测指标并运行

设置好在线人数、消息频率、群规模等指标后启动压测。

## 查看压测报告

![压测报告](/images/advance/report.png)

### 报告字段解释

测试指标

表示当前压测机正在执行的压测参数。

测试报告

- 运行时长：开始压测后的持续时间
- 在线人数：当前模拟的在线用户数
- 离线人数：模拟用户里已经掉线的人数
- 发送消息：累计发送的消息数
- 发送速率：当前发送速率
- 发送成功：服务端确认成功的消息数
- 发送错误：服务端确认失败的消息数
- 接收消息：订阅端实际收到的消息总数
- 接收速率：当前消息投递速率
- 发送平均延迟：消息发出到服务端响应的平均耗时
- 接收平均延迟：消息发出到订阅者收到消息的平均耗时

## 报告分析

### 发送速率

发送速率可以近似理解为发送消息并发数，用来判断系统在高峰期能承受多大的发消息压力。

估算公式：

```text
峰值并发数 = DAU × 峰值活跃用户比例 × 每用户发送速率
```

示例：

- DAU：100000
- 峰值在线比例：15%
- 每用户发送速率：0.1 条/秒

则：

`峰值发消息并发数 = 100000 × 0.15 × 0.1 = 1500 条/秒`

### 接收速率

接收速率更适合衡量群聊、广播这类一发多收场景。

估算公式：

```text
峰值接收消息并发数 = 峰值活跃群数量 × 每群平均在线成员数量 × 每群消息发送频率
```

### 成功率与延迟

- 发送成功率越高，说明系统越健康
- 发送和接收平均延迟越小，说明系统越稳定
- 发送成功消息数与期望收到消息数一致时，说明没有漏消息

### 带宽指标

发送流量和接收流量可以直接反映服务器和网络需要承受的带宽压力。

## 常见问题

### 长连接上不去，达到几千后连接数不再增加

先检查 WuKongIM 和 nginx 的文件描述符限制。

查看当前限制：

```bash
ulimit -n
```

临时修改：

```bash
ulimit -n 100000
```

如果需要永久修改，可以编辑 `/etc/security/limits.conf`：

```text
* soft nofile 100000
* hard nofile 100000
```

同时把 nginx 的 `worker_connections` 调高到例如 `40960` 左右。
