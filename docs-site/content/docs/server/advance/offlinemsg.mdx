## 离线消息同步流程

`WuKongIM` 采用读扩散模式。  
如果你需要先理解读扩散和写扩散的区别，可以先参考这篇背景文章：

- [什么是读扩散和写扩散](https://blog.csdn.net/m0_53246313/article/details/122674197)

开启最近会话后，`WuKongIM` 会在服务端为每个用户维护一份属于自己的最近会话列表。

应用从离线变成在线后，第一步不是直接拉某个会话的消息，而是先同步最近会话列表。

![离线消息同步流程](/images/advance/offlinemsg1.png)

## 在线后的维护方式

应用在线后，应该持续在线维护最近会话列表，而不是每次进入页面再全量重拉。

这样做的目的有两个：

- 保证会话顺序、未读数和最新消息摘要始终是实时的
- 减少每次打开聊天页时的额外请求和状态修正

## 进入聊天界面后做什么

当用户点击某个会话进入聊天界面时，再同步该会话的消息内容。

同时需要把最近会话里的红点数清空，例如调用 `/conversations/setUnread`。

![会话消息同步](/images/advance/offlinemsg2.png)

## 推荐的接入顺序

1. 应用上线后先同步最近会话列表。
2. 在线状态下持续维护最近会话。
3. 用户进入具体频道后，再同步该频道消息。
4. 完成消息同步后，清理对应会话未读数。
