这页解决服务端集成最先会遇到的四个问题：

1. 服务端 API 应该从哪里访问
2. 这些 API 适合暴露到哪里
3. 客户端连接地址怎么获取
4. 用户 token 和服务端鉴权各自负责什么

## 一句话先说明职责边界

- **客户端 token**：给客户端登录和连接 WuKongIM 用
- **服务端 API**：给你的业务系统管理消息、用户、频道、会话等能力用
- **路由地址**：告诉客户端应该连接哪个 TCP / WebSocket 地址

## API 访问入口

默认 HTTP API 监听在：

```text
http://<host>:5001
```

当前仓库里的默认示例配置也是这个端口，例如 `config/wk.yaml` 和 `exampleconfig/single.yaml`。

## 安全原则

服务端 API 应视为**业务内网接口**，不建议直接暴露到公网。

推荐做法：

- 只让业务服务访问 `5001`
- 在网关、防火墙或内网环境中隔离
- 管理后台和 API 分开考虑权限边界

## 服务端鉴权怎么理解

当前仓库里能看到三层相关配置：

- `tokenAuthOn`
  控制客户端连接时是否校验用户 token
- `managerToken`
  如果配置了这个值，HTTP API 请求需要在请求头里带 `token`
- `auth.superToken` / `auth.kind` / `jwt`
  用于更完整的认证配置，适合服务端或管理侧集成

这意味着新文档不应该再简单写成“API 完全无需认证”。更准确的说法是：

**API 默认用于内网服务调用，具体是否要求认证取决于你的部署配置。**

## 客户端连接地址怎么拿

客户端真正连接的不是 `5001`，而是 TCP 或 WebSocket 地址。通常有两种方式：

### 方式一：直接使用你明确配置的对外地址

例如：

- TCP：`<host>:5100`
- WebSocket：`ws://<host>:5200`

### 方式二：通过路由接口获取

路由接口：

```text
GET /route
```

它会返回：

- `tcp_addr`
- `ws_addr`
- `wss_addr`

当你需要根据不同部署形态动态下发连接地址时，这个接口更合适。

## 用户 token 接口做什么

`POST /user/token` 用于更新某个用户在某个设备上的 token 信息，常用于：

- 登录后给客户端下发连接凭据
- 重新登录或刷新 token
- 多设备区分主设备和从设备

也就是说：

- **你的业务系统负责生成 uid / token**
- **WuKongIM 负责使用这些凭据建立连接和维护会话**

## 建议的接入顺序

1. 先确认 API 只在受控网络中开放
2. 再确认你的鉴权配置是 `managerToken`、`superToken` 还是其他认证方式
3. 给客户端准备 `uid` 和 `token`
4. 通过固定地址或 `/route` 下发连接地址
5. 再继续看消息、用户、频道、会话等接口
