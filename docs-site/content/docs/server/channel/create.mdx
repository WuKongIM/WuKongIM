创建频道对应的核心接口是：

```text
POST /channel
```

## 这个接口适合做什么

- 新建群聊
- 初始化业务频道
- 创建频道时顺便写入初始成员

## 最重要的参数

| 参数 | 必填 | 说明 |
| --- | --- | --- |
| `channel_id` | 是 | 频道唯一 ID |
| `channel_type` | 是 | `1` 个人频道，`2` 群组频道 |
| `subscribers` | 否 | 初始订阅者列表 |
| `ban` | 否 | 是否全员禁言 |
| `send_ban` | 否 | 是否禁止发送消息 |
| `allow_stranger` | 否 | 个人频道是否允许陌生人发消息 |

## 最小示例

```bash
curl -X POST "http://localhost:5001/channel" \
  -H "Content-Type: application/json" \
  -d '{
    "channel_id": "group123",
    "channel_type": 2,
    "subscribers": ["user1", "user2", "user3"]
  }'
```

## 使用建议

- `channel_id` 由你的业务系统统一生成和管理
- 先明确频道是否是个人频道还是群组频道
- 如果创建时已经知道成员，直接传 `subscribers`
- 如果成员关系会频繁变化，可以先建频道，再走成员管理接口
